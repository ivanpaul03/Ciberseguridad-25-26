## 1. Información General

- **Fecha del escaneo:**  11/02/2026
- **Herramienta utilizada:** Nmap  
- **Versión:**  7.98
- **IP objetivo:**  192.168.56.101

---

## 2. Descubrimiento del Host

- **Estado del host:** Activo (Host is up)  
- **Latencia detectada:** 0.00045s (aproximadamente, dentro de red local)  
- **Método de descubrimiento utilizado:** Descubrimiento de hosts mediante ICMP Echo Request (ping scan) utilizando el comando:

```bash
nmap -sn 192.168.56.101/24
```

---

## 3. Sistema Operativo Detectado

- **Sistema operativo identificado:** Linux (Kernel 2.6.X – probable Ubuntu/Debian antiguo)  
- **Nivel de precisión estimado:** 98% de exactitud  
- **Método de detección:** Detección de sistema operativo mediante fingerprinting TCP/IP utilizando el comando:

```bash
sudo nmap -O 192.168.56.101
```

---

## 5. Enumeración de Servicios

- **Servicios detectados:** FTP, SSH, Telnet, HTTP (Apache), SMB y MySQL.  
- **Configuraciones relevantes:**  
  - Servicio FTP con acceso anónimo habilitado.  
  - Servicio Telnet activo (protocolo inseguro sin cifrado).  
  - Servicio SMB expuesto en los puertos 139 y 445.  
  - Servidor web Apache en versión desactualizada.  
- **Métodos de autenticación:**  
  - FTP: Usuario/contraseña (permite acceso anónimo).  
  - SSH: Autenticación mediante usuario y contraseña.  
  - MySQL: Autenticación mediante credenciales.  
  - SMB: Autenticación basada en usuarios del sistema.  

---

## 6. Usuarios y Autenticación

- **Usuarios detectados:** Enumeración de usuarios mediante scripts SMB.  
- **Acceso anónimo permitido:** Sí, en el servicio FTP.  
- **Contraseñas vacías detectadas:** No se detectaron contraseñas vacías, aunque se identificaron credenciales débiles en algunos servicios.

---

## 7. Vulnerabilidades Detectadas

- **Vulnerabilidades encontradas (CVE si aplica):**  
  - CVE-2011-2523 (vsftpd 2.3.4 Backdoor).  
  - Vulnerabilidades conocidas en Samba 3.0.20.  
- **Nivel de riesgo:** Alto.  
- **Descripción técnica:**  
  Se detectaron servicios con versiones obsoletas que presentan vulnerabilidades conocidas públicamente. Estas podrían permitir ejecución remota de código, escalada de privilegios o acceso no autorizado al sistema.

---

## 8. Servicios Web

- **Puertos web abiertos:** 80/TCP  
- **Tecnologías detectadas:** Apache 2.2.8 sobre sistema Linux  
- **Métodos HTTP permitidos:** GET, HEAD, POST, OPTIONS  
- **Información relevante del servidor:**  
  Servidor web sin cifrado HTTPS habilitado. Versión antigua con posibles vulnerabilidades conocidas.

---

## 9. Conclusiones

- **Nivel de exposición del sistema:** Alto.  
- **Principales riesgos detectados:**  
  - Servicios inseguros activos.  
  - Software desactualizado con vulnerabilidades conocidas.  
  - Acceso anónimo habilitado en FTP.  
  - Múltiples servicios innecesariamente expuestos a la red.  

- **Recomendaciones:**  
  - Actualizar servicios desactualizados.  
  - Cerrar puertos innecesarios.  
  - Configurar firewall adecuadamente.  
  - Deshabilitar autenticaciones débiles.  
  - Implementar HTTPS en el servidor web.
